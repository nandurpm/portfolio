DEPLOYMENT INSTRUCTIONS
================================================================================

Quick Deployment (5 minutes):

1. Create a GitHub repository:
   - Go to github.com and create a new repository
   - Name it "portfolio"
   - Make it public

2. Upload files:
   - Option A: Use Git command line
   - Option B: Use GitHub Desktop
   - Option C: Drag and drop on GitHub web interface

3. Enable GitHub Pages:
   - Go to Settings → Pages
   - Select "main" branch
   - Click Save

4. Your site is now live at:
   - https://yourusername.github.io/portfolio

5. (Optional) Add custom domain:
   - Enter your domain in GitHub Pages settings
   - Update DNS records with your domain provider
   - Wait for DNS propagation (15 minutes to 24 hours)

For detailed instructions, see: DEPLOYMENT_GUIDE.md

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

1. Site not loading?
   - Check GitHub Pages is enabled
   - Clear browser cache
   - Wait for deployment (1-2 minutes)

2. Styles not showing?
   - Verify CSS file paths
   - Clear browser cache
   - Check browser console for errors

3. Custom domain not working?
   - Verify DNS records
   - Check CNAME file exists
   - Wait for DNS propagation (up to 24 hours)

4. Links not working?
   - Verify relative paths are correct
   - Check for typos in file names
   - Test locally first

For more help, see: DEPLOYMENT_GUIDE.md
